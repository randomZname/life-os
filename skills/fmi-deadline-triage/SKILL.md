# Триаж на ФМИ срокове

Цел: от списък предстоящи задачи (от `fmi_get_upcoming_deadlines` / календар) →
подреден план кое да се прави първо и кога.

## Стъпки

1. Събери всичко: домашни, тестове, контролни, проекти + датите им.
2. За всяко оцени:
   - **Спешност** — колко дни остават.
   - **Тежест** — тест/контролно/проект > обикновено домашно.
   - **Усилие** — груба оценка часове.
3. Подреди по композитен приоритет: близък срок + висока тежест = най-горе.
4. Предложи конкретен план: какво днес, какво тази седмица, какво може да чака.

## Правила

- Не пропускай нищо със срок до 48 ч — то винаги е най-горе.
- Голям проект с далечен срок → разбий на стъпки, започни рано.
- Ако два срока се застъпват → започни този с по-голямо усилие по-рано.
- Бъди реалистичен с часовете; не предлагай 14-часов ден.

## Изходен формат

```
🔴 ДНЕС (спешно)
• [курс] — [задача] · срок [ДД.ММ HH:MM] · ~[N]ч

🟡 ТАЗИ СЕДМИЦА
• [курс] — [задача] · срок [ДД.ММ] · ~[N]ч

🟢 ПО-НАТАТЪК (започни рано)
• [курс] — [задача] · срок [ДД.ММ]

Предложение за днес: [1–3 конкретни действия]
```
